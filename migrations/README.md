# Migrations Directory

This directory is auto-generated and managed by Prisma. It contains the incremental SQL scripts (`migration.sql` files) that track the history of changes made to the database schema over time.

## Purpose

Prisma creates a new folder in this directory every time a migration is generated using the `npm run db:migrate` command (or `npx prisma migrate dev`). These scripts are used to:
1. Version-control exactly how the database has evolved.
2. Allow new developers to set up a fresh database that exactly mirrors the current schema state by recursively applying these migrations history files.

**Note:** You generally do not need to edit the files in this directory manually. Manage your database schema by editing `prisma/schema.prisma` and running Prisma's migration CLI tools.
