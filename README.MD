## рџ”Ќ Overview

This service provides a secure, scalable backend for a UPI-like digital payment platform. It handles user authentication, account management, and real-time fund transfers with concurrency safety, PIN verification, and rate limiting baked in.

**Key capabilities:**
- JWT-based user authentication with bcrypt password hashing
- UPI PIN verification layer for sensitive transactions
- Atomic fund transfers with balance integrity checks
- Immutable transaction history logging
- Rate limiting to mitigate abuse and brute-force attacks
- Schema-validated request payloads via Zod and express-validator

## рџљЂ Getting Started

### Prerequisites

Ensure the following are installed on your system:

- [Node.js](https://nodejs.org/) `>= 18.x`
- [npm](https://www.npmjs.com/) `>= 9.x`
- [Docker Desktop](https://www.docker.com/products/docker-desktop/) (for PostgreSQL)

---

### Installation

```bash
# Clone the repository
git clone https://github.com/your-org/upi-transaction-backend.git
cd upi-transaction-backend

# Install dependencies
npm install
```

---

### Environment Variables

Create a `.env` file in the project root by copying the example:

```bash
cp .env.example .env
```

Then configure the following values:

| Variable | Description | Example |
|---|---|---|
| `PORT` | Port the server listens on | `3000` |
| `DATABASE_URL` | PostgreSQL connection string | `postgresql://postgres:postgres@localhost:5433/upi_transactions?schema=public` |
| `JWT_SECRET` | Secret key for signing JWTs | `your_strong_secret_here` |

> вљ пёЏ **Never commit your `.env` file.** It is listed in `.gitignore` by default.

---

### Database Setup

**1. Start the PostgreSQL container:**

```bash
docker-compose up -d
```

**2. Generate the Prisma client:**

```bash
npm run db:generate
```

**3. Apply database migrations:**

```bash
npm run db:migrate
```

**4. (Optional) Seed the database with test data:**

```bash
npm run db:seed
```

---

### Running the Server

```bash
# Development mode (with hot-reload via Nodemon)
npm run dev

# Production mode
npm start
```

The server will be available at `http://localhost:3000`.

## рџ”ђ Security

This service implements multiple layers of security:

- **JWT Authentication** вЂ” All protected endpoints require a valid Bearer token in the `Authorization` header.
- **UPI PIN Verification** вЂ” Sensitive transaction endpoints require an additional PIN check via `authPinMiddleware`.
- **Rate Limiting** вЂ” Endpoints are throttled to prevent brute-force and DoS attacks.
- **Password Hashing** вЂ” All passwords are hashed with `bcrypt` before storage; plaintext passwords are never persisted.
- **Input Validation** вЂ” All incoming request bodies are validated with Zod and express-validator before reaching any controller.
- **Concurrency Safety** вЂ” Fund transfer logic handles concurrent requests to prevent double-spend and race conditions.
